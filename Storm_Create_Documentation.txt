STORM CREATE — ПОЛНАЯ ПОЛЬЗОВАТЕЛЬСКАЯ ДОКУМЕНТАЦИЯ
Актуально на: 16.03.2026  (редакция 1.5)

============================================================
0) ДЛЯ КОГО ЭТОТ ДОКУМЕНТ
============================================================
Эта инструкция написана для пользователей платформы:
- владельцев ботов;
- администраторов дочерних ботов;
- менеджеров магазина;
- пользователей, которые впервые создают бота через Storm Create.


============================================================
1) ЧТО ТАКОЕ STORM CREATE
============================================================
Storm Create — это конструктор Telegram-ботов.

Через интерфейс платформы можно:
- создать Shop-бота (интернет-магазин);
- создать VPN-бота;
- создать FileStore-бота (раздача файлов по ссылкам);
- принимать оплаты (Stars, CryptoBot, ЮMoney и т.д.);
- управлять заказами и клиентами;
- запускать рассылки;
- вести продажи прямо в Telegram.


============================================================
2) ВАЖНЫЕ РОЛИ НА ПЛАТФОРМЕ
============================================================
1. Владелец (Owner)
Создаёт бота, управляет ключевыми настройками и доступами.

2. Администратор дочернего бота
Работает в /admin внутри конкретного бота.

3. Покупатель
Оформляет заказ, оплачивает, получает товар/чек.


============================================================
3) БЫСТРЫЙ СТАРТ ДЛЯ НОВОГО ПОЛЬЗОВАТЕЛЯ
============================================================
1. Откройте главного бота Storm Create.
2. Нажмите “Создать бота”.
3. Выберите тип бота: Shop, VPN или FileStore.
4. Подключите токен из BotFather.
5. После создания откройте дочернего бота.
6. Введите /admin и настройте его (для Shop и VPN).
   Для FileStore-бота: подключите канал и загрузите первый файл.


============================================================
4) КАК ПОЛУЧИТЬ API-ТОКЕН БОТА В BOTFATHER
============================================================
Пошагово:
1. Откройте @BotFather.
2. Введите /newbot.
3. Укажите имя бота.
4. Укажите username (должен заканчиваться на "bot").
5. Получите токен вида:
   123456789:AA...
6. Скопируйте токен и отправьте его в мастер создания Storm Create.

Важно:
- Токен — это пароль вашего бота.
- Никому его не передавайте.
- Если токен утек, в BotFather используйте /revoke и получите новый.


============================================================
5) КАК СОЗДАТЬ БОТА В STORM CREATE (ПОЛНЫЙ ТУТОРИАЛ)
============================================================
1. В главном боте нажмите “Создать бота” или /newbot.
2. Выберите тип:
   - Интернет-магазин
   - VPN-бот
3. Введите название бота.
4. Вставьте API-токен, полученный в BotFather.
5. При запросе укажите администратора (или оставьте себя).
6. Дождитесь подтверждения “Бот создан”.
7. Откройте ссылку на нового бота.
8. Для настройки введите /admin.

Если токен не принимается:
- проверьте, не вставили ли пробел в начале/конце;
- проверьте, не используется ли токен другим ботом;
- создайте новый токен в BotFather.


============================================================
6) ГЛАВНЫЙ БОТ: ОСНОВНЫЕ РАЗДЕЛЫ
============================================================
Обычно в главном боте используются:
- Создать бота
- Мои боты
- Профиль
- Премиум
- Маркетплейс
- Помощь

В “Профиле” доступны:
- ваш ID;
- баланс;
- пополнение;
- история операций.


============================================================
7) КАК РАБОТАТЬ С "МОИ БОТЫ"
============================================================
Раздел “Мои боты” позволяет:
- открыть карточку каждого бота;
- перейти к настройкам;
- запустить/остановить;
- удалить;
- продлить подписку (если требуется тариф).

Совет:
после создания сразу перейдите в настройки бота и проверьте платежи до запуска рекламы.


============================================================
8) ПЕРВАЯ НАСТРОЙКА SHOP-БОТА (ЧЕК-ЛИСТ)
============================================================
После открытия /admin внутри Shop-бота:
1. Настройки → Текст приветствия — написать приветственное сообщение.
2. Категории → добавить минимум 1 категорию.
3. Товары → добавить первые товары с фото и ценой.
4. Оплата → включить нужные методы и проверить тестовым заказом.
5. Чеки → настроить канал для копий чеков.
6. Онбординг (/admin → Настройки → 🚀 Онбординг) — проверить прогресс чек-листа.
7. Пройти весь путь покупателя: Каталог → Корзина → Оплата → Чек.


============================================================
9) КАТЕГОРИИ И ТОВАРЫ: ПОЛНЫЙ ТУТОРИАЛ
============================================================
9.1 Добавление категории
1. /admin -> Категории.
2. Нажмите “Создать категорию”.
3. Укажите название (и, при необходимости, фото/описание).

9.2 Добавление товара
1. /admin -> Товары -> “Добавить товар”.
2. Заполните шаги: название, описание, цена, фото, категория.
3. Выберите тип товара.

Типы товара:
- physical — физический;
- digital_auto — цифровой с авто-выдачей;
- digital_manual — цифровой с ручной выдачей админом.

9.3 Остатки и ключи
- Для обычного товара задаётся остаток.
- Для digital_auto можно использовать цифровые ключи/контент для выдачи.


============================================================
10) ПЛАТЕЖИ В SHOP-БОТЕ: ПОДРОБНО
============================================================
Путь: /admin -> Платежи

Можно включать/выключать:
1. Telegram Stars
2. CryptoBot
3. ЮMoney
4. QR/Перевод

10.1 Telegram Stars
- Включите Stars в BotFather.
- Включите Stars в админке платежей.

10.2 CryptoBot
- Должен быть настроен токен у владельца платформы.
- В админке дочернего бота просто включите метод.

10.3 ЮMoney
- Укажите кошелек в настройках платежей.
- Токен API берется из платформы.

10.4 QR/Перевод (manual_qr)
- Введите ссылку на QR-код или текст реквизитов для перевода.
- При оплате пользователь нажимает "Я перевел".
- Бот ОБЯЗАТЕЛЬНО запрашивает скриншот или файл-квитанцию.
- Без файла/скриншота заказ не уходит на подтверждение.
- Подтверждение автоматически уходит администратору.

============================================================
11) КАК ПОДКЛЮЧИТЬ ЧАТ/КАНАЛ ДЛЯ КОПИЙ ЧЕКОВ
============================================================
Это именно настройка “чеков”, о которой вы просили.

Путь:
1. /admin -> Чеки (или “Чеки и отчеты”).
2. Нажмите кнопку “📢 Чат: ...” (настройка канала/чата чеков).
3. Введите:
   - ID чата/канала, например: -1001234567890
   или
   - username, например: @mychannel
4. Убедитесь, что бот добавлен в этот чат/канал.

Отключение:
- В этом же поле отправьте: off

Если чат/канал чеков не подключен:
- копии чеков отправляются администраторам бота автоматически.

Дополнительно в разделе чеков:
- включение/выключение блоков чека (владелец, поддержка, партнеры);
- стили чеков и предпросмотр (в тарифах Pro/Ultra).


============================================================
12) КАК ПОДКЛЮЧИТЬ КАНАЛ ДЛЯ АВТОПОСТИНГА
============================================================
Это отдельная функция от “чеков”.

Путь:
1. /admin -> Настройки -> “📢 Конфигурация канала”.
2. Нажмите “Установить канал”.
3. Введите ID или @username канала.
4. Бот отправит тестовое сообщение.

Обязательно:
- бот должен быть администратором канала;
- у бота должно быть право отправлять сообщения.

После подключения:
- настройте события для автопубликации (новые товары, заказы, отзывы и т.д.);
- можно делать ручные публикации прямо из админки.


============================================================
13) ОБЯЗАТЕЛЬНАЯ ПОДПИСКА НА КАНАЛ
============================================================
Путь:
1. /admin -> Настройки -> “🔒 Обязательная подписка”.
2. Установите ID канала.
3. Установите ссылку на канал (t.me/...).
4. При необходимости измените текст сообщения (доступ зависит от тарифа).

Важно:
- бот должен быть админом канала, иначе проверка подписки работать не будет.

Отключение:
- в этом же разделе есть кнопка отключения.


============================================================
14) АВТОСООБЩЕНИЕ (НАПОМИНАНИЯ О НЕАКТИВНОСТИ)
============================================================
Путь:
1. /admin -> Настройки -> "🔔 Автосообщение".
2. Включите функцию.
3. Установите время в часах (от 1 до 720), через сколько часов бездействия отправлять напоминание.
4. Отредактируйте текст сообщения.

Для чего:
- возвращать неактивных пользователей обратно в бот.


============================================================
15) РАЗДЕЛ "ПОМОЩЬ" ВНУТРИ БОТА
============================================================
Путь:
1. /admin -> Настройки -> “❓ Раздел Помощь”.
2. Добавляйте статьи с заголовком и текстом.
3. Включайте форму обратной связи.

Что получает пользователь:
- удобное FAQ-меню прямо в боте;
- возможность написать в поддержку.


============================================================
16) РАССЫЛКИ
============================================================
Путь:
1. /admin -> Рассылки.
2. Отправьте текст/медиа сообщения.
3. При необходимости добавьте кнопку.
4. Подтвердите отправку.

Рекомендация:
- отправьте сначала тест на себя/админов;
- не отправляйте слишком длинные сообщения без структуры.


============================================================
17) ЗАКАЗЫ И СТАТУСЫ: КАК ЭТО РАБОТАЕТ
============================================================
Для покупателя:
1. Открыть каталог.
2. Добавить товар в корзину.
3. Оформить заказ.
4. Выбрать оплату.
5. Получить чек/статус.

Для админа:
1. Получить уведомление о новом заказе.
2. Если QR/реквизиты — проверить присланный скрин/файл оплаты.
3. Подтвердить оплату.
4. При digital_manual — выдать товар вручную.

При digital_auto:
- выдача может происходить автоматически после подтверждения оплаты.


============================================================
18) ПОПОЛНЕНИЕ БАЛАНСА ПОЛЬЗОВАТЕЛЯ
============================================================
В главном боте:
1. Профиль -> Пополнить баланс.
2. Введите сумму.
3. Выберите способ оплаты.
4. После подтверждения баланс обновится.

В дочернем Shop-боте:
1. Профиль -> Пополнить баланс.
2. Введите сумму.
3. Выберите доступный метод.
4. Подтвердите оплату.

Важно:
- настройки пополнения общего баланса управляются платформой;
- админы дочерних ботов не меняют эти реквизиты по своему усмотрению.

21) ИНТЕГРАЦИИ: СДЭК И МОЙСКЛАД
============================================================
Путь: /admin → 🔗 Интеграции

Доступные интеграции:

21.1 СДЭК (расчёт стоимости доставки API)
Позволяет автоматически рассчитывать стоимость доставки при оформлении заказа.

Как подключить:
1. Войдите в личный кабинет СДЭК (https://lk.cdek.ru/).
2. Перейдите в Настройки → Доступ к API.
3. Скопируйте Account и Secure Password.
4. В боте введите в формате: Account:Secure_Password

После подключения при выборе доставки СДЭК покупатель вводит город,
и бот автоматически рассчитывает стоимость.

21.2 МойСклад (синхронизация остатков)
Позволяет автоматически синхронизировать остатки товаров с системой МойСклад.

Как подключить:
1. Войдите в МойСклад (https://online.moysklad.ru/).
2. Перейдите в Настройки → Пользователи и права → Токены.
3. Создайте токен и скопируйте его.
4. В боте введите токен.

После подключения остатки товаров обновляются автоматически при подтверждении заказов.

Если интеграция не работает:
- проверьте корректность данных (Account:Password для СДЭК, токен для МойСклад);
- убедитесь в корректности прав доступа в личном кабинете сервиса.


============================================================
22) ЧТО ДЕЛАТЬ, ЕСЛИ /ADMIN НЕ ОТКРЫВАЕТСЯ
============================================================
1. Убедитесь, что вы администратор именно этого дочернего бота.
2. Проверьте, что бот запущен.
3. Попробуйте /start, потом /admin.
4. Если не помогло — обратитесь к владельцу платформы.


============================================================
23) ЧАСТЫЕ ОШИБКИ И БЫСТРЫЕ РЕШЕНИЯ
============================================================
Ошибка: Payment_provider_invalid
Что значит:
- невалидный токен платежного провайдера карт.
Что делать:
- выбрать Stars/CryptoBot/ЮMoney;
- сообщить владельцу платформы.

Ошибка: “Платеж не найден”
Что делать:
- подождать и нажать “Проверить” повторно;
- проверить корректность метода оплаты.

Ошибка: бот не может писать в канал
Что делать:
- добавить бота админом канала;
- дать право отправлять сообщения;
- проверить ID/username канала.

Ошибка: “токен уже используется”
Что делать:
- сгенерировать новый токен в BotFather.

- начать заново через /start -> Купить VPN.

Проблема: “Не удалось восстановить данные платежа”

Решение:
- повторить оплату из свежего меню;
- при необходимости подтвердить заказ вручную через /admin.

Проблема: trial не активируется
- сработал anti-abuse или trial уже использован.
Решение:
- проверить настройки trial в /admin;
- проверить возраст аккаунта в anti-abuse.

Проблема: пользователь не видит “Инструкции”/“Статус”
- функция выключена в “🚀 Возможности”.
Решение:
- включить соответствующий toggle.

Проблема: в strict_webhook оплата не выдается
- нет подтверждения webhook.
Решение:
- проверить webhook-интеграцию и secret;
- как временный обход подтвердить заказ вручную.

44) ФЛЭШ-РАСПРОДАЖИ
============================================================
Временная скидка на весь каталог или отдельные товары с таймером обратного

Параметры:
- Скидка (%)          — от 1 до 90%
- Длительность (ч)    — от 1 до 720 часов
- Список товаров      — ID через запятую, или пусто = весь каталог
- Лейбл               — произвольный текст, например «🔥 ФЛЭШ-СКИДКА»

2. Нажмите «▶️ Запустить сейчас».

   - лейбл со скидкой и зачёркнутой ценой;
   - таймер «⏱ Осталось: Xч Yм».
4. По истечении времени скидка отключается автоматически.

Как остановить досрочно:
- Нажмите «🛑 Остановить» в том же разделе.

Покупатель видит:
  🔥 ФЛЭШ-СКИДКА  -20%
  💰 Цена со скидкой: 800 руб.  ~~1000~~
  ⏱ Осталось: 2ч 15м

47) ПОДАРОЧНЫЕ СЕРТИФИКАТЫ
============================================================
Покупатель может приобрести электронный сертификат на выбранную сумму
и передать код другу. Получатель вводит код при оформлении заказа.

Путь: /admin → Настройки → ⚡ Доп. функции → 🎁 Подарочные сертификаты

Настройка (для администратора):
- Вкл/Выкл
- Минимальная сумма сертификата (₽)
- Максимальная сумма сертификата (₽)
- Ручная выдача: вы сами генерируете код на нужную сумму без оплаты

Как работает для покупателя:
1. Выбирает «Подарочный сертификат» в меню.
2. Вводит желаемую сумму в пределах диапазона.
3. Оплачивает выбранным методом.
4. Получает код вида GIFT-XXXXXXXX.
5. Передаёт код получателю любым способом.

Как получатель использует сертификат:
- В корзине → «Промокод» → вводит код GIFT-XXXXXXXX.
- Сумма сертификата автоматически вычитается из итога заказа.
- Сертификат одноразовый.

Ручная выдача администратором:
1. Откройте раздел «Подарочные сертификаты».
2. Нажмите «🎁 Выдать сертификат вручную».
3. Введите сумму → получите готовый код.

49) РАСШИРЕННАЯ АНАЛИТИКА
============================================================
Углублённая статистика продаж поверх базовой.

Путь: /admin → 📊 Аналитика → 📊 Расш. аналитика
  или: /admin → Настройки → ⚡ Доп. функции → 📊 Расширенная аналитика

Разделы:

49.1 Тепловая карта активности
Показывает количество оплаченных заказов:
- По каждому часу (0:00 — 23:00) в виде гистограммы.
- По каждому дню недели (Пн–Вс).
- Выделяет пиковый час и пиковый день.

Используйте для: планирования рассылок и флэш-акций в часы пик.

49.2 Когортный анализ
Разбивает покупателей по неделе регистрации и показывает:
- Сколько человек зарегистрировалось.
- Конверсия (% купивших хотя бы раз).
- Удержание (% купивших 2 и более раз).

49.3 Воронка продаж
Пошаговая воронка от посетителя до повторной покупки:
  Посетители → Добавили в корзину → Начали заказ → Оплатили → Повторные

Каждый шаг показывает абсолютное число и % конверсии от первого шага.

49.4 Автотеги клиентов
Система автоматически разбивает базу на 4 сегмента:
- 👑 VIP          — сумма покупок от 10 000 ₽
- ✅ Активные     — есть заказ за последние 30 дней
- 😴 Спящие       — нет активности более 60 дней
- 🆕 Новые        — остальные

Для каждого сегмента показывается список пользователей (до 5 в превью).
Используйте с функцией сегментированных рассылок.

54) РЕЙТИНГ МАГАЗИНА

Панель администратора:
- Текущий средний рейтинг и общее число оценок.
- Распределение по звёздам (1★ … 5★).

- Голосуют только покупатели с минимальным числом оплаченных
  заказов (настраивается, по умолчанию 1).
- Один голос на пользователя (или с интервалом 24 ч при смене).

Отображение в /start:
- Показывается, если набрано минимум 3 оценки.
  Пример: ⭐⭐⭐⭐⭐ 4.8 / 5  (27 оценок)

----------------------------------------------------------

3. (Опционально) Напишите описание файла или нажмите «Пропустить».

- Видео

- Аудио

1. Пользователь переходит по ссылке t.me/<бот>?start=file_<хеш>.
2. Бот автоматически отправляет файл в ЛС.
3. Если задано описание и/или превью — они показываются перед файлом.

- Название и описание (превью 25 символов)
- Кнопка ⚙️ для перехода в карточку файла

Карточка файла позволяет:
- Посмотреть текущую ссылку для раздачи
- ✏️ Изменить описание (введите «-» чтобы удалить)
- 🖼 Заменить или удалить превью
- 🗑 Удалить файл из списка

65.5 ТЕХНИЧЕСКИЕ ОСОБЕННОСТИ
----------------------------------------------------------
- Файлы хранятся в виде Telegram file_id (без физической копии на сервере).
- Данные о файлах сохраняются в filestore_data.json (не в основной БД).
- Один FileStore-бот поддерживает неограниченное количество файлов.
- Один подключённый канал/чат на бота.
- Ссылки не истекают — работают до момента удаления файла администратором.

Важно:
- Если файл удалён из Telegram (истёк срок хранения на серверах Telegram),
  бот не сможет его выдать — пользователь увидит сообщение об ошибке.
- Загружать файлы может только администратор бота (владелец).
